## 一、作品创意理念

### 1.1 创作目的

针对高校师生日常校园咨询需求，打造专属智能对话助手，解决以下痛点：

- **信息分散**：校园通知、办事流程、规章制度等信息分散在多个平台
- **咨询不便**：线下咨询时间受限，电话咨询效率低下
- **重复劳动**：教务、后勤等部门重复回答相同问题

### 1.2 作品简介

「校园校小助」是基于 Coze 平台开发的校园专属智能对话工作流，采用「知识库优先 + 联网兜底」双引擎架构，提供以下核心功能：

| 功能模块  | 说明                  |
| ----- | ------------------- |
| 知识库检索 | 优先检索校园专属知识库，确保信息准确  |
| 智能追问  | 问题模糊时主动澄清，提升交互体验    |
| 办事导航  | 生成详细办事指南，包含步骤、材料、地点 |
| 网页跳转  | 识别跳转需求，经确认后跳转校园系统   |
| 相关推荐  | 回答后推荐相关问题，提升使用效率    |

### 1.3 创新点

1. **优先级机制**：本地知识库优先检索，无结果再联网搜索，确保校园信息准确性
2. **模块化架构**：六大功能节点独立设计，便于维护和功能扩展
3. **高适配性**：支持网页内嵌、上下文记忆，无缝对接校园官网
4. **可视化配置**：提供 UI 配置界面，降低使用门槛

### 2.2 工作流节点

| 节点名称                      | 功能描述  | 输入                | 输出                                  |
| ------------------------- | ----- | ----------------- | ----------------------------------- |
| knowledge\_search\_node   | 知识库检索 | user\_query       | knowledge\_result, knowledge\_score |
| intent\_recognition\_node | 意图识别  | user\_query       | intent\_type                        |
| smart\_clarify\_node      | 智能追问  | user\_query       | clarify\_questions                  |
| answer\_generation\_node  | 答案生成  | knowledge\_result | response\_content                   |
| service\_guide\_node      | 办事导航  | user\_query       | guide\_steps                        |
| jump\_confirm\_node       | 跳转确认  | intent\_type      | jump\_url                           |
| web\_search\_node         | 联网搜索  | user\_query       | web\_search\_result                 |
| response\_optimize\_node  | 话术优化  | response\_content | optimized\_content                  |
| related\_recommend\_node  | 相关推荐  | response\_content | related\_questions                  |
| no\_result\_node          | 无结果回复 | -                 | default\_response                   |

## 四、配置指南

### 4.1 知识库配置（核心）

#### 方式一：配置文件配置（推荐）

编辑 `config/knowledge_config.json`：

```json
{
  "knowledge_base_id": "kb_xxxxxxxxxx",
  "knowledge_base_name": "校园校小助知识库",
  "top_k": 3,
  "min_score": 0.5,
  "enabled": true,
  "description": "校园校小助知识库配置"
}
```

**参数说明**：

| 参数                  | 说明         | 默认值  |
| ------------------- | ---------- | ---- |
| knowledge\_base\_id | 知识库 ID（必填） | 空    |
| top\_k              | 检索返回条数     | 3    |
| min\_score          | 最低相关度阈值    | 0.5  |
| enabled             | 是否启用知识库    | true |

#### 方式二：环境变量配置

```bash
# Windows PowerShell
$env:COZE_KNOWLEDGE_BASE_ID="kb_xxxxxxxxxx"

# macOS / Linux
export COZE_KNOWLEDGE_BASE_ID="kb_xxxxxxxxxx"
```

#### 方式三：Coze 平台绑定

在 Coze IDE 的工作流设置中直接绑定知识库。

### 4.2 模型配置

编辑 `config/` 目录下的 LLM 配置文件：

```json
{
  "model": "doubao-seed-2-0-lite-260215",
  "temperature": 0.3,
  "max_tokens": 2000
}
```

**推荐配置**：

| 场景    | temperature | 说明   |
| ----- | ----------- | ---- |
| 知识库回答 | 0.1-0.3     | 稳定准确 |
| 智能追问  | 0.3-0.5     | 适度灵活 |
| 相关推荐  | 0.5-0.7     | 创意推荐 |

### 4.3 功能开关配置

通过 UI 界面（`ui/config.html`）或代码配置：

| 开关                   | 说明      | 默认 |
| -------------------- | ------- | -- |
| knowledge\_first     | 知识库优先模式 | 开启 |
| web\_search\_enabled | 联网搜索兜底  | 开启 |
| jump\_enabled        | 网页跳转功能  | 开启 |
| clarify\_enabled     | 智能追问    | 开启 |
| recommend\_enabled   | 相关推荐    | 开启 |

## 七、注意事项

### 7.1 知识库配置

- ⚠️ **知识库 ID 必须正确**：错误的 ID 会导致检索失败
- ⚠️ **知识库内容要丰富**：建议上传完整的校园文档
- ⚠️ **定期更新知识库**：确保信息时效性

### 7.2 部署环境

- ⚠️ **Python 版本**：需要 Python 3.12 或更高版本
- ⚠️ **网络环境**：确保能访问 Coze 平台
- ⚠️ **API 配额**：注意 LLM 调用次数限制

### 7.3 功能限制

- ⚠️ **知识库优先**：优先检索知识库，无结果才联网
- ⚠️ **跳转确认**：网页跳转需用户确认，不会自动跳转
- ⚠️ **上下文记忆**：依赖会话管理，刷新页面会丢失

### 7.4 常见问题

| 问题       | 解决方案                  |
| -------- | --------------------- |
| 知识库检索无结果 | 检查知识库 ID、确认知识库有内容     |
| 模型调用失败   | 检查 API 配额、网络连接        |
| 工作流执行异常  | 查看 Coze IDE 日志、检查节点配置 |
