# 校园校小助 UI 界面产品需求文档

## 1. 产品概述

### 产品名称
**校园校小助管理后台** (Campus Assistant Admin Dashboard)

### 产品定位
基于 Coze 平台的可视化管理界面，用于展示校园智能问答工作流的运行原理、节点配置，并提供便捷的工作流链接和管理功能。

### 目标用户
- 校园信息化管理人员
- Coze 平台运维人员
- 开发人员查看工作流结构

---

## 2. 功能需求

### 2.1 工作流可视化展示

#### 功能描述
以图形化方式展示校园校小助工作流的完整流程，包括：
- 10个功能节点的拓扑关系
- 节点间的数据流向
- 条件分支逻辑
- 节点的输入输出定义

#### 展示内容
1. **流程图展示**
   - 使用现代流程图库（如 Mermaid.js 或 GoJS）
   - 支持缩放、平移操作
   - 支持节点高亮和详情查看

2. **节点信息卡片**
   - 节点名称和类型（Task/Agent/Condition）
   - 功能描述
   - 使用的配置文件
   - 输入输出字段

3. **流程路径**
   - 意图分支：web_jump / service_guide / campus_consult / other
   - 知识库结果分支：knowledge_answer / web_search
   - 搜索结果分支：optimize_response / no_result

#### 优先级
**P0** - 必须实现

---

### 2.2 模块配置管理

#### 功能描述
提供工作流模块的可视化配置界面，包括：
- LLM 模型配置
- 知识库参数设置
- 网页跳转链接管理
- 功能开关（联网搜索、网页跳转等）

#### 配置项

1. **知识库配置**
   - 知识库 ID
   - top_k 参数（默认 3）
   - min_score 阈值（默认 0.5）

2. **LLM 模型配置**
   - 模型名称
   - temperature
   - max_completion_tokens
   - 系统提示词
   - 用户提示词模板

3. **网页跳转配置**
   - 跳转链接列表管理
   - 添加/编辑/删除链接
   - 关键词匹配规则

4. **功能开关**
   - 联网搜索开关
   - 网页跳转开关
   - 知识库优先模式

#### 优先级
**P1** - 重要功能

---

### 2.3 Coze 工作流链接

#### 功能描述
提供快速链接到 Coze 平台的入口：
- Coze IDE 链接
- 工作流测试入口
- 知识库管理入口

#### 链接项
- Coze 工作流编辑器
- Coze 知识库管理
- Coze API 配置
- 部署状态监控

#### 优先级
**P0** - 必须实现

---

### 2.4 工作流测试入口

#### 功能描述
提供快速测试工作流的界面：
- 输入测试问题
- 选择用户角色（student/teacher）
- 查看输出结果
- 查看执行路径

#### 优先级
**P2** - 增强功能

## 4. 设计规范

### 4.1 视觉风格

**设计方向**：科技感 + 简洁专业

**配色方案**：
- 主色：`#2563EB` (蓝色 - 科技感)
- 辅色：`#1E40AF` (深蓝)
- 强调色：`#10B981` (绿色 - 成功)
- 警告色：`#F59E0B` (橙色)
- 错误色：`#EF4444` (红色)
- 背景色：`#F8FAFC` (浅灰白)
- 侧边栏：`#1E293B` (深灰蓝)
- 文字：`#334155` (深灰)

**字体**：
- 标题：`"Noto Sans SC", "PingFang SC", sans-serif`
- 正文：`"Inter", "Noto Sans SC", sans-serif`
- 代码：`"JetBrains Mono", monospace`

**圆角**：
- 卡片：`12px`
- 按钮：`8px`
- 输入框：`6px`

**阴影**：
- 卡片阴影：`0 4px 6px -1px rgba(0,0,0,0.1)`
- 悬浮阴影：`0 10px 15px -3px rgba(0,0,0,0.1)`

### 4.2 动效规范

**页面切换**：
- 淡入淡出：`opacity 0→1, 300ms ease-out`

**卡片悬浮**：
- 缩放：`scale(1.02), 200ms`
- 阴影加深：`0 10px 15px -3px rgba(0,0,0,0.1)`

**流程图节点**：
- 悬浮高亮：`filter: brightness(1.1), 150ms`
- 连接线动画：`stroke-dashoffset` 动画

### 4.3 响应式设计

**断点**：
- 桌面：`≥1024px` - 完整侧边栏
- 平板：`768px - 1023px` - 可折叠侧边栏
- 移动：`≤767px` - 底部导航

## 5. 技术方案

### 5.1 技术栈

**前端**：
- HTML5 + CSS3 + JavaScript
- Mermaid.js（流程图渲染）
- Prism.js（代码高亮）
- 纯前端，无需后端

**资源**：
- CDN 加载外部库
- 本地资源使用相对路径

### 5.2 文件结构

```
ui/
├── index.html              # 工作流概览页
├── nodes.html              # 节点详情页
├── config.html             # 模块配置页
├── links.html              # Coze链接页
├── test.html               # 测试页面
├── css/
│   └── style.css           # 全局样式
├── js/
│   ├── main.js             # 主逻辑
│   ├── flowchart.js        # 流程图渲染
│   └── config.js           # 配置管理
└── assets/
    └── images/             # 图片资源
```

### 5.3 数据来源

**工作流定义**：
- 从 `src/graphs/graph.py` 解析节点和连接
- 从 `src/graphs/nodes/` 读取节点代码获取描述
- 从 `config/*.json` 读取配置参数

**硬编码方式**：
- 为简化实现，直接在 JS 中定义工作流结构
- 便于维护和展示
